# Support-Ticket-System
Веб-система регистрации и обработки заявок технической поддержки с ролевой моделью доступа и AI-классификацией обращений через Google Gemini.

Overview

Система автоматизирует цикл работы с обращениями техподдержки: от создания заявки пользователем до закрытия специалистом и анализа статистики. Решает проблему небольших организаций, которым не нужна тяжёлая helpdesk-инфраструктура — всё приложение это один Node.js-процесс с файловой базой SQLite, запускаемый одной командой. Отличительная фича — описание заявки анализируется AI (категория, приоритет, рекомендации), а отдельный модуль прогнозирует время обработки на основе истории.

Features


Регистрация/вход по JWT, роли user и admin, у админов — специализация по категории (responsibleCategory)
Жизненный цикл заявки: новая → в работе → закрыта, с авторасчётом времени обработки
AI-анализ описания через Gemini (цепочка из 3 моделей с фоллбеком при ошибках) — категория, приоритет, рекомендации
Прогноз SLA: медианное время закрытия похожих заявок из истории
Поиск/фильтрация заявок, оценка закрытых заявок (1–5), удаление своих заявок
Очередь администратора: смена статуса, ручная корректировка категории/приоритета, подсветка просроченных заявок
Аналитика: диаграммы по статусам и категориям на чистом CSS, без сторонних библиотек
Управление аккаунтами, полный журнал истории изменений заявок


Architecture

Трёхуровневая архитектура. Представление — три статические HTML-страницы со своими JS-модулями и общим api.js, без фреймворков и сборки. Бизнес-логика — единый server.js (Express): middleware аутентификации/ролей и REST-маршруты по сущностям. Хранение — файл SQLite, доступ через промис-обёртки над sqlite3. AI-анализ — прямой REST-вызов Gemini API из сервера без SDK.
